Why a Bitcoin Full Node?

Bitcoin is a peer-to-peer network. In order to run efficiently and effectively, it needs peers....and the more the better.

This tutorial will describe how to create a "full node" (a Bitcoin service that contains the full blockchain and allows transactions to propagate throughout the Bitcoin network via peers). For a detailed explanation for why this is important you can read this article about Bitcoin full nodes.

Why a Raspberry Pi?

Raspberry Pi is an inexpensive computing hardware that generates little heat, draws little power, and can run "in the basement" silently without having to think about it.

Background

I decided to create my own bitcoin full node on a Raspberry Pi. I feel like it is being a good bitcoin citizen to run a full node. I wanted to use a Raspberry Pi because it is low cost, generates little heat, draws little power, and can run "in the basement" silently without having to think about it.

I am going to assume that if you are reading this to create your own Raspberry Pi bitcoin full node, then you already know a fair amount about linux, electronics, and running tools like SSH. If not, I am probably going to skip over some helpful steps, which you might be irritating.

The guide listed above is quite good. Here are some things that would have made it easier.

64 GB MicroSD card: The blockchain is growing quickly, so I felt 64 GB was a better size of storage. The card came formatted as exfat, instead of fat32. The SDformatter app they recommend only formatted as exfat, so what to do? Since I run a Mac, I just used the built-in Disk Utility and formatted as "MS-DOS (FAT)," which is really fat32. I chose to install NOOBS to make installing Raspberry Pi easier. NOOBS is kind of a boot manager/OS selector all in one that makes getting started with Raspberry Pi super easy. Judge me all you want, it's cool, but I went that route and it was as easy as advertised. In my opinion, don't install any of the wallet software. This is just a full node to help relay transactions, etc. It also makes the installation easier. Feel free to use one of the countless other free wallets out there. As you are running all of the apt-get commands in the tutorial - add this one to the end: sudo apt-get install libtool (you will need it and the main article doesn't cover it, but it is in the comments) When you clone the git respository, change the version to the current version (in my case 0.10) Conclusion

In the end, it only took about 1-2 hours to get everything set up and then another 36-48 hours to have the blockchain sync'd up.
